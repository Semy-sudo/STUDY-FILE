템플릿 엔진
웹 템플릿과 웹 컨텐츠 정보를 처리하기 위해 설계된 소프트 웨어
html 과 db connectio 분리

1)서버사이트 템플릿엔진: DB 혹은 API 에서 가져온 데이터를 미리 정의된
Template에 넣어 html 그려서 클라이언트에 전달해 주는 역할

클라이언트에게 요청받기->DB/API에서 정보 가져오기->적당한template에 끼우기
->서버에서 html만들기->html 클라이언트로 보내기
2)클라이언트사이드 템플릿 엔진
html 형태로 코드 작성할 수 있으며,DOM 그리게 해주는 역할

클라이언트에서 공통적프레임을 미리 template으로 만듬->서버에서 데이터 받기->
데이터를 template적절한 위치에 replace ->DOM 객체 그리기

템플릿엔진의 필요성
1.유지보수에 좋다-하나의 템플릿 만들어놓으면 여러 페이지 렌더링할수 있다
2.코드간결
3.재사용성 높다-대부분 웹페이지는 똑같은 디자인페이지에 데이터만 바뀜

@WebServlet 이란
속성값을 통해 해당 Servlet과 매핑될 url패턴 지정
